5 core patterns, mental models, recurrences, and templates

🧭 Pattern Decision Framework

Signal: single index, linear state
→ 1D DP (dp[i])
Signal: grid, two strings/sequences
→ 2D DP (dp[i][j])
Signal: items with weight/value, capacity
→ Knapsack DP
Signal: longest common/increasing subsequence
→ LCS / LIS DP
Signal: subarray/substring, merge intervals
→ Interval DP (dp[i][j])

Pattern 1 — 1D DP (Linear State)

Mental Model: Each cell dp[i] answers: "What is the best answer considering the first i elements?" You only look back at a fixed number of previous states. Think of it as climbing stairs — each step depends on a few steps behind you.
Climbing Stairs / Fibonacci recurrence dp[i] = dp[i-1] + dp[i-2]
# base cases: dp[0] = 1, dp[1] = 1
python — climbing stairs template 
def climbStairs(n):
    if n <= 1: return 1
    dp = [0] * (n + 1)
    dp[0], dp[1] = 1, 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

# Space-optimized (O(1) space):
def climbStairsOpt(n):
    a, b = 1, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b

Pattern 2 — 2D Grid DP

Mental Model: dp[i][j] answers: "What is the best answer to reach cell (i, j)?" You can only come from the top or left (for path problems). Fill the table row by row. The answer is always at dp[m-1][n-1].
Unique Paths / Min Path Sum recurrence dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])
# base case: first row and first column filled with prefix sums
python — min path sum template 
def minPathSum(grid):
    m, n = len(grid), len(grid[0])
    dp = [[0]*n for _ in range(m)]
    dp[0][0] = grid[0][0]
    for i in range(1, m): dp[i][0] = dp[i-1][0] + grid[i][0]
    for j in range(1, n): dp[0][j] = dp[0][j-1] + grid[0][j]
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])
    return dp[m-1][n-1]

Pattern 3 — 0/1 Knapsack

Mental Model: For each item, you make a binary choice: take it or leave it. dp[i][w] = max value using first i items with capacity w. The key insight: if you take item i, you use up weight[i] capacity, so you look back at dp[i-1][w - weight[i]].
0/1 Knapsack recurrence dp[i][w] = max(dp[i-1][w], # skip item i
                    dp[i-1][w - wt[i]] + val[i]) # take item i
python — 0/1 knapsack template 
def knapsack(weights, values, capacity):
    n = len(weights)
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for w in range(capacity + 1):
            dp[i][w] = dp[i-1][w]  # skip
            if weights[i-1] <= w:
                dp[i][w] = max(dp[i][w],
                    dp[i-1][w - weights[i-1]] + values[i-1])
    return dp[n][capacity]

# Coin Change variant (unbounded — item reuse allowed):
def coinChange(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for coin in coins:
        for w in range(coin, amount + 1):
            dp[w] = min(dp[w], dp[w - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

Pattern 4 — LCS / LIS (Subsequence DP)

Mental Model (LCS): dp[i][j] = length of LCS of s1[:i] and s2[:j]. If characters match, extend the diagonal. If not, take the best of skipping one character from either string.

Mental Model (LIS): dp[i] = length of LIS ending at index i. For each i, scan all j < i where nums[j] < nums[i] and extend.
LCS recurrence if s1[i-1] == s2[j-1]: dp[i][j] = dp[i-1][j-1] + 1
else:                  dp[i][j] = max(dp[i-1][j], dp[i][j-1])
python — LCS + LIS templates 
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0]*(n+1) for _ in range(m+1)]
    for i in range(1, m+1):
        for j in range(1, n+1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]

def lis(nums):  # O(n^2) — easy to remember
    n = len(nums)
    dp = [1] * n
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)

Pattern 5 — Interval DP

Mental Model: dp[i][j] = best answer for the subproblem on the range [i, j]. You iterate by length of the interval (from small to large), and for each interval you try every possible split point k. The answer is always dp[0][n-1].
Interval DP recurrence (e.g. Matrix Chain / Burst Balloons) dp[i][j] = min/max over all k in [i, j-1]:
                dp[i][k] + dp[k+1][j] + cost(i, k, j)
python — interval DP template (burst balloons) 
def maxCoins(nums):
    nums = [1] + nums + [1]  # pad boundaries
    n = len(nums)
    dp = [[0] * n for _ in range(n)]

    for length in range(2, n):       # interval length
        for i in range(0, n - length):  # left boundary
            j = i + length               # right boundary
            for k in range(i+1, j):     # last balloon to burst
                coins = nums[i] * nums[k] * nums[j]
                dp[i][j] = max(dp[i][j],
                    dp[i][k] + dp[k][j] + coins)
    return dp[0][n-1]
